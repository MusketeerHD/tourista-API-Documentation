This is the documentation for tourista API

API for Tourista, an travel app by Six Kizuki, built using Node.js, Express, MySQL.

  • Author: Six Kizuki

Features

Here are some features of the Tourista API based on the provided code :

  1. User Authentication :
    • Register a new user: POST router.post('/auth/register', Register);
    • User login: POST router.post('/auth/login', Login);
    • Refresh authentication token: GET router.get('/auth/token', refreshToken);
    • Logout user: DELETE router.delete('/auth/logout', Logout);
  2. User Profile Management :
    • Get user profile by ID: GET router.get("/users/:id", verifyToken, getUserProfileById);
    • Create user profile: POST router.post('/users/createProfile', verifyToken, createUserProfile);
    • Update user profile: PUT router.put('/users/updateProfile', verifyToken, updateUserProfile);
    • Delete user profile: DELETE router.delete('/users/deleteProfile', verifyToken, deleteUserProfile);
    • Upload user profile image: POST router.post("/users/uploadProfile", verifyToken, uploadFile);
  3. Authorization:
    • Verify token: Middleware 'verifyToken' used in routes

These features allow users to register, login, manage their profiles, and perform actions such as creating, updating, and deleting their profiles. Additionally, there is token-based authentication implemented to verify user authorization for protected routes.

Prerequisites and Technology

Before diving into the integration process, make sure you have the following prerequisites and technologies in place:

  • Express JS

    • Express is a fast and minimalist web application framework for Node.js. It provides a robust set of features for building web APIs. Ensure you have Express installed and configured for your project.

  • MySQL 2

    • Mysql2 is a MySQL database driver for Node.js. It allows you to connect your application to a MySQL database and perform database operations. Install and set up the Mysql2 package to interact with your MySQL database.

  • Sequelize

    • Sequelize is an Object-Relational Mapping (ORM) library for Node.js. It provides a convenient way to interact with databases using JavaScript objects. Integrate Sequelize into your project to simplify database operations and manage models and associations.

  • JSON Web Tokens (JWT)

    • JSON Web Tokens is a standard for token-based authentication. It enables secure communication between the client and server by generating and validating tokens. Utilize JWT to handle authentication and authorization in your API.

  • Dotenv

    • Dotenv is a module that loads environment variables from a .env file into Node.js applications. It allows you to store sensitive information like API keys or database credentials securely. Set up a .env file and configure it to load the necessary environment variables.

  • Bcrypt

    • Bcrypt is a library used for hashing and salting passwords in Node.js. It adds an extra layer of security to user authentication by encrypting passwords before storing them in the database. Install and implement Bcrypt to securely store and verify user passwords.

  • Cookie Parser

    • Cookie Parser is a middleware for Express that simplifies handling HTTP cookies in your application. It parses incoming request cookies and makes them accessible through the request object. Integrate Cookie Parser to manage and manipulate cookies in your API.

  • CORS

    • Cross-Origin Resource Sharing (CORS) is a mechanism that allows restricted resources on a web page to be requested from another domain outside the domain from which the resource originated. Implement CORS to handle cross-origin requests in your API.

  • Nodemon

    • Nodemon is a development tool that automatically restarts your Node.js application whenever file changes are detected. It provides a convenient way to test and debug your API during the development process. Install and configure Nodemon for a smoother development experience.

Make sure you have the latest versions of these technologies installed and properly configured in your project environment. They will play a crucial role in integrating Tourista API seamlessly into your application.

Installation

Firstly, you need to clone this repository using this following command :

git clone https://github.com/msandypr/tourista-api.git

Then, open the project folder using cd jwt-auth-express, depends on your clone folder location, after that, you need to install dependency using this following command :

npm install or yarn install or pnpm install

Run & Usages

After that, run the jwt-auth-express locally using nodemon, if you never install nodemon, you should install first using npm install -g nodemon or using yarn global add nodemon if you using yarn. If you already install nodemon, please use this following command :

nodemon index

This will run the application locally on http://localhost:5000. Dont forget to create database first on your local machine, db name is auth_db or change it by yourself in config/Database.js

Auth API

This is the Auth API endpoint for the tourista and run locally on http://localhost:5000

Register

This endpoint is used to register a new user.

  • URL = '/auth/register'
  • Method = POST
  • Request Body =
    Parameter Type Required Description
    'username' 'STRING' Yes The desired username
    'email' 'STRING' Yes The email address of the user
    password 'STRING' Yes The password for the user
    confirmPassword 'STRING' Yes Confirmation password for the user
  • Response Format =
    1. If Response Success : "msg": "Register Success!"
    2. If Email was Registered : "msg": "Email already Registered!"
    3. If Password not match with Confirm Password : "msg": "Password and Confirm Password doesn't match."
    4. If Response Fail : "msg": "Forbidden"

Login

This endpoint is used to log in a user.

  • URL = '/auth/login'
  • Method = POST
  • Request Body =
    Parameter Type Required Description
    'email' 'STRING' Yes The email address of the user
    'password' 'STRING' Yes The password for the user
  • Response Format =
    1. If Response Success : "msg": "Login Successful"
    2. If Email is not Registered or Wrong : "msg": "Email not Registered! or Wrong Email!"
    3. If Password not match : "msg": "Wrong Password!"
    4. If Response Fail : "msg": "Forbidden"

RefreshToken

This endpoint is used to refresh the access token.

  • URL = '/auth/token'
  • Method = GET
  • Authentication = Refresh Token
  • Response Format :
    1. If Response Success : "Example" "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoibXVza2V0ZWVyIiwiZW1haWwiOiJtdXNrZXRlZXJAZ21haWwuY29tIiwiaWF0IjoxNjg2MDUyODMxLCJleHAiOjE2ODYxMzkyMzF9.gNgZ6tajVl9OVDUYcHp5IatNx_yJj7V-yScDXDVdhW0"

Logout

This endpoint is used to log out a user and revoke the access token.

  • URL = '/auth/logout'
  • Method = DELETE
  • Authentication = Refresh Token
  • Response Format =
    1. If Response Success : "msg": "Logout Successful"

Tourista Table Models

The Tourista API utilizes various models to organize and manage data related to different entities within the application. Here are some details about each model :

DestinationDetail.js

  • This model represents the detailed information of a destination. It includes data such as descriptions, attractions, and other relevant details.

DestinationModelDetail.js

  • The DestinationDetailModel is responsible for handling the management of destination detail data. It provides methods for creating, retrieving, updating, and deleting destination details.

DestinationPenginapan.js

  • The DestinationPenginapan model stores information about penginapan (accommodation) options available at a particular destination. It includes details like the name, location, facilities, and pricing of various accommodations.

DestinationPenginapanModel.js

  • This model handles the operations related to managing destination penginapan data. It provides functions for creating, updating, retrieving, and deleting penginapan information.

DestinationWisata.js

  • The DestinationWisata model represents different attractions and points of interest at a destination. It stores data such as the name, description, location, and other relevant information about each wisata.

DestinationWisataModel.js

  • The DestinationWisataModel is responsible for managing destination wisata data. It includes functions for creating, updating, retrieving, and deleting wisata information.

ReviewPenginapan.js

  • The ReviewPenginapan model handles reviews and feedback for different penginapan options. It allows users to share their experiences and ratings for accommodations.

ReviewPenginapanModel.js

  • This model provides functionalities to manage reviews of penginapan. It includes methods for adding new reviews, retrieving existing reviews, and performing other review-related operations.

ReviewWisata.js

  • The ReviewWisata model facilitates user reviews and ratings for various wisata. It enables users to share their opinions and experiences regarding different attractions.

ReviewWisataModel.js

  • The ReviewWisataModel manages the reviews and related operations for wisata. It includes functions for adding new reviews, retrieving existing ones, and performing other review-related tasks.

Trip.js

  • The Trip model represents a trip within the Tourista application. It contains information such as the destination, duration, itinerary, and other relevant details related to a user's travel plan.

TripModel.js

  • The TripModel is responsible for managing trip data. It provides functionalities for creating new trips, updating existing ones, retrieving trip details, and handling other trip-related operations.

Umkm.js

  • The Umkm model represents Micro, Small, and Medium Enterprises (UMKMs) within the Tourista application. It stores information about different businesses, including their names, locations, contact details, and other relevant data.

UmkmDetail.js

  • The UmkmDetail model contains detailed information about an UMKM. It includes data such as the business description, products/services offered, operating hours, and other relevant information.

UmkmDetailModel.js

  • The UmkmDetailModel is responsible for managing the detailed information of UMKMs. It provides methods for creating, updating, retrieving, and deleting UMKM details.

UmkmModel.js

  • The UmkmModel handles the management of UMKM data within the Tourista API. It includes functionalities for creating new UMKMs, updating existing ones, retrieving UMKM information, and performing other UMKM-related operations.

UserModel.js

  • The UserModel represents a user within the Tourista application. It stores user-related information such as usernames, email addresses, and encrypted passwords.

UserProfileModel.js

  • The UserProfileModel manages user profile data. It includes functionalities for creating and updating user profiles, retrieving profile information, and handling other profile-related tasks.

These models, along with their corresponding model files, form the backbone of the Tourista API, ensuring efficient management and retrieval of data related to destinations, accommodations, attractions, reviews, user profiles, and more.
